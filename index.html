Git Commands

Setup

  1. git config --global user.name "username"
  2. git config --global user.email "email@example.com"

Push Code to GitHub

  1. git init
  2. git add <filename>
  3. git status
  4. git commit -m "<message>"
  5. git branch -M main
  6. git remote add origin <URL>
  7. git push -u origin main

.gitignore

The .gitignore file is a fundamental component in Git for managing version control by specifying intentionally untracked files and directories that Git should ignore. It prevents these specified items from being included in the repository's history and from appearing as "untracked files" in git status.

Commands

Git Restore

  • git restore [options] <file> — Restore file in working directory
  • git restore --staged <file> — Unstage file
  • git restore --source <commit_hash> <file> — Restore from specific commit

git restore is a Git command used to undo changes in your working directory or staging area (index). It helps you restore files to a previous state — like undoing edits or un-staging files.

Git Revert

  • git revert <commit_hash> — Create new commit that reverses target commit
  • git revert HEAD — Undo the last commit
  • git log — Find the commit hash first

git revert is a Git command used to undo a commit by creating a new commit that reverses the changes of a previous commit. Unlike git reset, it does not remove history; it safely undoes changes without rewriting commit history.

Git Reset

  • git reset <commit_hash> — Move HEAD to specified commit (default: --mixed mode)
  • git reset --soft <commit_hash> — Keep changes in staging area
  • git reset --mixed <commit_hash> — Keep changes in working directory but unstage them
  • git reset --hard <commit_hash> — Discard all changes completely
  • git reset HEAD~1 — Undo the last commit

git reset is a Git command used to undo commits by moving the current branch pointer to a different commit. Unlike git revert, it rewrites commit history by removing commits. It's powerful but should be used carefully, especially on shared branches, as it can cause issues for collaborators.

Git Branches

  • git branch — List all local branches
  • git branch <branch_name> — Create a new branch
  • git branch -d <branch_name> — Delete a branch (safe delete)
  • git branch -D <branch_name> — Force delete a branch
  • git checkout <branch_name> — Switch to an existing branch
  • git checkout -b <branch_name> — Create and switch to a new branch
  • git switch <branch_name> — Modern way to switch branches
  • git switch -c <branch_name> — Create and switch to a new branch
  • git merge <branch_name> — Merge specified branch into current branch
  • git branch -M <new_name> — Rename current branch

Git branches are lightweight pointers to commits that allow you to work on different features, bug fixes, or experiments in isolation. They enable parallel development where multiple people can work on different parts of a project simultaneously without interfering with each other's work. Branches are a core feature of Git's distributed version control system.
